逻辑工坊 · 积木手册

猜条件 是一款解谜闯关游戏。每一关都有一个隐藏条件，运行示例程序时该条件会悄悄生效。比较示例的“真实答案”和运行后看到的结果，你就能推断出本关的条件是什么；再为预测题选择 T / F / U / I，全对即可通关。

工作台主要使用三类积木：

- 指令积木：放在顶层，作为一条指令执行。
- 判断积木：放在 [block:answer] 回答 的六边形孔里，决定本问的结论。
- 变量积木：放在圆角孔里，提供一个候选值（数字 / 字符串 / 变量）。

变量名通过 [value:$名称] 这样的形式引用；运行后能在右侧“变量状态”面板查看。

指令积木

[block:answer] **回答** — 执行六边形里的判断，把结果（T 是 / F 否 / U 不确定 / I 无法回答）作为本问的答复，追加到回答面板。

[block:assign] **赋值** — 把圆角孔里的值存入变量名；后续可用 [value:$名称] 引用该变量。

[block:repeat] **重复** — 把花括号里的指令重复执行指定次数（最多 64 次，防止死循环）。每次执行都共享同一份变量。

[block:declareRelation] **声明…关系** — 声明一个关系名，让它出现在变量视图的「关系」列表里。游戏自带的「大于/等于/小于/含有」默认已声明，再次声明无副作用。

[block:overrideRelation] **覆写…为…** — 把 (左值, 关系名, 右值) 的判断结果强制改为 是 / 否 / 不确定 / 无法回答 之一；可以覆写游戏自带的关系，不会触发连锁反应（例如 覆写 (1) 大于 (2) 为 (否) 不会让 (2) 大于 (3) 也变成 否）。

判断积木（放进 [block:answer] 的六边形孔）

[block:compare] **比较** — 单个积木带 4 个运算符的下拉选框：大于 / 等于 / 小于 / 含有。「大于/小于/含有」只接受数值；「等于」兼容同类型；「含有」按子串判断。

[block:not] **不成立** — 把内部判断的 T / F 翻转，U 与 I 保持不变。

[block:isQuestion] **第 N 问** — 回放第 N 个历史回答的结果。仅在 [block:answer] 中可用。

[block:relationCheck] **关系判断** — 查询 (左值, 关系名, 右值) 的关系结果：先看是否被「覆写」过；再判断关系名是否与游戏自带的 大于/等于/小于/含有 同名（同名则走内置实现）；否则返回 无法回答。适合搭配「声明关系 / 覆写关系」构建自定义推理。

变量积木（放进圆角孔）

[block:value] **值 / 变量** — 在圆角孔里直接输入：纯数字视为整数或浮点数，带引号的视为字符串，其他文本默认是字符串；以 [value:$] 开头的视为变量引用。

[block:random] **随机变量** — 取 [起点, 终点] 区间内的整数作为候选值。

[block:arithmetic] **四则运算** — 对左右两个候选域做 加 / 减 / 乘 / 除以，结果再作为下游的候选值。

[block:answerValue] **…的答案** — 把判断结果转成字符串（是 / 否 / 不确定 / 无法回答），方便参与赋值或后续比较。

[block:answerCharCount] **…的字数** — 返回判断文字的字数（忽略空格、括号、$ 等符号）。

[block:currentLine] **行号** — 当前回答在工作区里的行号。

[block:currentQuestion] **问题编号** — 当前回答是第几个问题。

T / F / U / I 的含义

- T（是） — 所有候选组合下，命题都成立。
- F（否） — 所有候选组合下，命题都不成立。
- U（不确定） — 真与假并存，例如随机变量的某些取值让命题为真、另一些为假。
- I（无法回答） — 求值过程出错，例如除零、未赋值的变量、参数类型不匹配。

猜条件玩法

每一关卡片都包含：

1. 示例程序：工作台上已经放好的积木块，运行后观察回答。
2. 预测题：关卡卡片下方的若干小题，每个小题都有一个 [block:answer] 回答 程序，你需要在 T / F / U / I 中选出预期的答案。

全部预测正确即可解锁下一关。提示：示例程序的输出通常已经暴露了本关条件；多对比几个判断的“真值”和实际输出，就能锁定规律。

积木文字表示

所有积木的文字表示唯一合法，但忽略全角 / 半角符号差异：例如 `(3)` 与 `（3）` 等价，`[回答 ...]` 与 `【回答 ...】` 等价。仓库里默认填充的文字（圆角括号里）可以替换。
